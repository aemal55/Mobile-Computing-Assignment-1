Sketches

Sketch 1 depicts the loading page, which is the app's main opening view after the user launches it; it includes user-friendly design elements such as the logo and name.

  1. When a user enters the app and clicks, the loading page appears for a fraction of a second before jumping to the landing page.

Sketch 2 represents the landing page, where the majority of the app's interaction with the user will take place.

  1. The user will see two languages, Russian and English, from which they can translate the text and convert English to Russian depending on their preference.
  2. A voice feature will be added to provide definitive responses from text to speech for a more user-friendly experience, assisting in converting Russian texts from English into an automated voice function that will be able to voice out the words in the text to the user in speech.

Sketch 3 is a continuation of Sketch 2 and will depict the same interaction between the user, but this time in the opposite direction, with English to Russian instead of Russian to English.

  1. In Sketch 2, the user can translate from Russian to English. However, the opposite can be done, such as allowing users to convert English to Russian alongside the voice feature.

Sketch 4 depicts the "Favourites" screen, where all of the user's selected interactions will be saved for later viewing if the user so chooses.

  1. If the user translates 4 texts from Russian to English, it will allow the user to select all or a few selected 2-3 texts by clicking on the heart shaped icon displayed near the translate box in Sketches 2 and 3 as previously described on the top right corner edge where the user can favourite the text and close the app to return to it later to review the text the user converted will be saved.
  2. If there are number of texts then the user is also able to utillize the use of scrollbar to scroll up and down.

Sketch 5 depicts the "Favourites" deletion screen, where the app interaction, which included the text marked as "Favourites," can be deleted by selecting the text.

  1. The user will be able to select and delete favourites based on which ones are required to be kept and which are not.
  2. If the user wishes to delete the selected "Favourites," the user may do so by clicking the "Edit" button. Once clicked, the user will be shown crossbars in red, which allow the user to delete all at once or delete a few and keep the rest.
  3. When the user deletes the favourites, the app will alert the user stating "Are you sure you want to delete? Yes or No "whether the user wishes to delete or not.

Sketch 6 depicts the "History" screen, which will save the user's entire text history if the user does not want to lose important text.

  1. If the user wants to keep all of the translated conversions they will be kept in history.
  2. If the user wants to view the previous texts, the user may do so by checking the history.
  3. To check history the user will click on the navigation menu bar below the screen to click on the history button to see the previous texts.

The "History" deletion screen, seen in Sketch 7, shows how all previous text messages are stored. If the user chooses, the user can choose to keep or erase the whole text history.

  1. Depending on which entries must be retained and which may be deleted, the user will be able to choose and delete history.
  2. Clicking the "Edit" button will allow the user to remove the selected "History" if they so want. When the user clicks, red crossbars will appear, giving the option to delete everything at once or select a small number of texts to remove while keeping the rest.
  3. Upon deletion of the history the app will send the user a alert stating "Are you sure you want to delete? Yes or No" whether the user wants to delete or not.

When the user clicks on the gear icon in Sketch 8, the user will be taken to the "Settings" screen. This screen shows settings for maintaining history, rating the application, and additional current-related apps.

  1. A button that the user can switch on or off to store or not save history will be visible on the settings screen.
  2. Few more buttons will be shown to the user, 2 more to be precise to either rate the app or see apps related to the current one.
